The CashierLoginPage component is responsible for rendering the UI for the cashier to log in. It displays a form with an input for the employee's name, a readonly input for the PIN, and a numpad for the cashier to enter their PIN. When the cashier submits the form, the component makes a POST request to the API to verify the credentials. If the response is successful, the component redirects the cashier to the cashier route.

Ingredients()

The Ingredients component manages and displays a list of ingredients. It allows users to add, update, delete, and sort ingredients. The component fetches ingredient data from an API and manages the editing state for adding or updating ingredients. Features: - Fetches ingredients from an API and displays them in a table. - Allows sorting by various fields (ID, name, quantity, price, threshold, restock). - Provides a dialog for adding or editing ingredients with fields for name, quantity, price, and threshold. - Handles adding, updating, and deleting ingredients via API calls. - Displays toast notifications for success or error scenarios. - Highlights ingredients that need restocking based on their threshold.

(async) handleCancel(id) → {Promise.<void>}

Handles the cancellation of an order by making a POST request to the server. If the request is already in progress, it does nothing. It sets the `isProcessing` state to true before the request and false after. If the request fails, it logs an error message. If the request succeeds, it updates the `orders` state by changing the status of the order to 'cancelled'. If the new status is 'cancelled', it removes the order from the list after a fade-out animation.
